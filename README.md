# MassiveDyno UML Canvas

大きなPlantUML図を、ブラウザだけで安全に描画・確認できる日本語ビューアーです。

**公開サイト:** https://plantuml.massivedyno.com

## 特長

- PlantUMLコードを貼り付けてブラウザ内で描画
- MarkdownのPlantUMLコードブロックにも対応
- ホイールによる拡大・縮小、ドラッグ移動、全体表示、全画面表示
- 操作バーを好きな位置へドラッグ移動
- PNG / SVGで保存
- 描画エラーを選択・コピー可能なテキストとして表示
- 大きすぎる図では、利用者が任意の描画上限を指定して再実行
- アクティビティ図のスイムレーン名を、縦に読み進めても追従表示
- アクティビティ図の箱を選択し、文章・主体・追加・複製・削除を画面上で微修正
- 箱を別の箱へドラッグ＆ドロップして前後を並べ替え
- 箱を横へドラッグして別の主体（スイムレーン）へ移動
- 「新しい箱」を図上へドラッグし、指定した箱の前後へ追加
- ドロップ前に移動先レーンまたは挿入位置を強調表示
- 選択した箱の後へ条件分岐を追加し、箱を「はい／いいえ」側へ移動
- 図上のIFひし形をクリックし、条件・分岐ラベルの変更、分岐解除、全体削除
- 図上のwhile条件もクリックして専用編集画面を表示
- 同じ直列フロー内で次の接続先を変更
- `while`の条件・継続・終了ラベルと、`fork`の枝を専用画面で編集
- 画面上の編集をPlantUMLコードへ反映し、元に戻す・やり直すにも対応
- 編集モードを開始した時点と現在のPlantUMLコードを、行番号付きの差分で確認
- 入力コードを外部のPlantUMLサーバーへ送信しないブラウザ内処理

## ローカルで動かす

### 必要環境

- Node.js 22.13以上
- npm

### 起動

```bash
git clone https://github.com/USS-Meigetsu/plantuml-viewer.git
cd plantuml-viewer
npm install
npm run dev
```

起動後、ターミナルに表示されるローカルURLをブラウザで開いてください。

`npm run dev` と `npm run build` の直前に、`@plantuml/core` からブラウザ実行資産を自動生成します。生成物はGit管理の対象外です。

## 主なコマンド

```bash
npm run dev       # 開発サーバー
npm run lint      # ESLint
npm test          # 本番ビルドと基本テスト
npm run build     # Sites向け本番ビルド
npm run test:pages # Cloudflare Pages向け静的ビルドと基本テスト
```

## Cloudflare Pages

`codex/cloudflare-pages` ブランチは、Cloudflare Pages向けの完全な静的サイトを
`out/` に出力します。Pagesのビルドコマンドは `npm run build:pages`、出力先は
`out` です。HTMLと画面用アセットはCloudflareのエッジから配信され、PlantUML
描画エンジンは利用者が初めて図を描画するときだけブラウザへ読み込まれます。

## 描画上限

初期上限は縦横それぞれ8,192pxです。上限超過時だけ変更UIが表示され、必要に応じて最大32,768pxまで指定できます。無制限にはせず、ブラウザのフリーズや過剰なメモリ消費を防いでいます。

## 箱の編集

アクティビティ図を描画した後に「箱を編集」を選ぶと、標準的な `:処理;` の箱を
図上または一覧から選択できます。文章変更、同じ制御構造内での前後移動、主体変更、
前後への追加、複製、削除が可能です。箱を別の箱の上半分・下半分へドラッグすると、
その箱の前・後へ移動します。編集のたびにPlantUMLコードへ反映して再描画し、
編集パネルと選択中の箱はそのまま維持されます。

ドラッグを始めた方向が縦なら順番変更、横なら主体変更として扱います。主体変更時は
移動先のスイムレーン全体を強調表示します。編集パネルの「新しい箱をドラッグ」は、
箱の上半分・下半分へ落とすことで、その前・後へ新しい処理を追加できます。

編集後は「コード差分」から、編集モードを開いた時点との差を確認できます。追加行は緑、
削除行は赤で表示し、巨大なコードでは変更箇所の前後だけを抜き出して表示します。

`if`、`while`、`fork`などの境界を越える並べ替えは、コード構造を壊さないよう制限しています。

## 構造の編集

箱を選択して「構造編集」を開くと、選択した箱の後への条件分岐追加、既存の
`if`の「はい／いいえ」側への移動、同じ直列フロー内での次の接続先変更ができます。
接続先変更は自由な線描画ではなく、PlantUMLで安全に表現できるよう箱の順序を
変更して実現します。

編集モードでは図上のIFひし形を直接クリックできます。条件と「はい／いいえ」ラベルの
変更に加え、分岐の枠だけを外して両側の箱を残す操作と、IF内の箱を含めた全体削除を
選べます。追加直後も同じ編集画面が開いたままになるため、その場で変更・削除できます。
削除操作も「元に戻す」の対象です。

「while・fork」では、標準的な`while ... endwhile`の条件・継続ラベル・終了ラベルを
編集できます。標準的な`fork / fork again / end fork`では、枝の追加・削除・順序変更が
可能です。入れ子や特殊な記法など、安全にコードへ戻せない構造は編集対象に含めません。

## プライバシー

PlantUMLコードと描画処理は利用者のブラウザ内に留まり、外部のPlantUML描画サーバーへ送信されません。入力内容は同じブラウザのローカルストレージにのみ保存されます。
